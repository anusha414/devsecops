what is application security? y is it important?
1. Application development lifecycle:- It has 3 important phases, dev:- developers work on source code and test locally, test:- qa enineers test unittests and regression testing or
  funtional testing, prod:- application is swicthed to production.
2. Above flow looks good but security is not taken into consideration, a bad actor can attack the application. This is where application security comes into picture.

SAST, DAST and SCA help in application security program.

SAST(static application securoty testing):-

1. Sonarqube is a popular tool, application is connected to sonarqube, sonarqube goes through the application and display if there are any security issues
2. Common cases sonarqube helps in resolving are below

SCA(software composition Analysis0:-
1. It looks at direct and indirect dependencies the application is using. for eg:- u might use a flask which is direct dependency but flask might be internallu using dependencies
    which are indirect and they might be prone to security threat.
2. Practically managing 1000's of indirect dependencies and 100's of direct dependencies is difficult.
3. every programming language has a SCA tools.

DAST(Dynamic application security testing):-

1. It is similar to SAST but only at runtime, SAST looks at source code of your application, but DAST uses url of our application and tests the application for security of
  application.
2. This is very important because this is how hackers work try to hack your application using 1000's of api calls to the application.
