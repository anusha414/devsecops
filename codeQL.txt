👉 CodeQL is like a very smart search engine for code.
Instead of:
“Search for this exact word”

It lets you ask questions like:
“Show me every place where user input goes directly into the database”
“Find code that looks safe but can actually be hacked”
“Are there risky patterns repeated across the entire project?”

CodeQL is a semantic code analysis engine that treats code as data and lets you query it to find security vulnerabilities and code quality issues.

There are 3 ways of setting CodeQL in your code:-

1. Enable default codeQL at global level, so it will be added to every repo.
2. Enable default codeQL at repo level, Settings->Advanced security->code scanning->codeQL Analysis
3. Enable custom codeQL by configuring a workflow in the repo.

NOTe:- whenever you enable default codeql it will detect for language and setup codeql accordingly
